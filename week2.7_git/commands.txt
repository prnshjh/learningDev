1. git init: Initializes a new Git repository in the current directory.

2. git clone [repository URL]: Copies a Git repository from a remote source to your local machine.

3. git add [file]: Adds a file or directory to the staging area.

4. git commit -m "[commit message]": Records changes to the repository with a descriptive message.

5. git status: Shows the status of changes as untracked, modified, or staged.

6. git diff: Shows the difference between the working directory and the staging area.

7. git diff --staged: Shows the difference between the staging area and the last commit.

8. git checkout [branch name or commit]: Switches branches or restores files from the working directory to a previous state.

9. git branch: Lists all local branches.

10. git branch [branch name]: Creates a new branch.

11. git merge [branch]: Combines changes from one branch into another.

12. git pull: Fetches changes from the remote repository and merges them into the current branch.

13. git push: Sends local commits to the remote repository.

14. git remote -v: Lists all remote repositories and their URLs.

15. git log: Displays a history of commits.

16. git reset [file]: Unstages a file while keeping its changes.

17. git reset --hard [commit]: Resets the current branch to a specific commit, discarding all changes after that commit.

18. git stash: Temporarily shelves changes so you can work on something else.

19. git stash pop: Applies the most recently stashed changes to the working directory.

20. git tag [tag name]: Creates a tag to mark a specific commit in history.
